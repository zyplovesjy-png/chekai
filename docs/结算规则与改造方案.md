## 一、比牌与筹码结算规则（定稿）

### 1.0 比牌时的喊价结构（由下注流程保证）

比牌开始时，存活玩家的最终喊价只可能是三种结构之一：

1. 无人敲：所有存活玩家最终喊价相等；
2. 有人敲：敲的玩家喊价可以少于统一值，其余存活玩家喊价相等；
3. 全员都敲：各自喊价可以不等。

### 1.1 两两比牌（现有 `compareSplit` 已正确，保留）

- 头尾都大，或一大一平 → 赢；
- 一头大一头小 → 平局，平局双方之间不发生任何筹码转移；
- 头尾完全相同 → 平局。

### 1.2 输家最终比价的分配（核心改动点一）

**最终比价** = 比牌时刻每位存活玩家的最终喊价（敲的玩家 = 敲出的簸簸数）。底注与芒果不算最终比价。

对每个输家 L（stake = L 的最终比价）：

1. 找出所有两两比牌中赢过 L 的玩家（winners）；
2. 将 winners 排序分梯队：
   - 尾牌大的在前；
   - 尾牌相同时头牌大的在前（赢同一个输家的人之间，尾牌相同则头牌必分胜负或完全相同）；
   - 头尾完全相同的玩家并列为同一梯队；
3. 按梯队顺序依次吃 L 的 stake：
   - 每个赢家从单个输家身上最多赢「自己的最终比价」（**按输家分别封顶**：同一赢家对每个输家的封顶各自独立计算，互不占用额度）；
   - 并列梯队内平分本梯队可吃的金额；若梯队内有人先到封顶，剩余由并列者继续吃；
   - 前一梯队全部封顶后，剩余传给下一梯队；
4. 所有赢家都封顶后仍有剩余 → **退还输家本人**；
5. 没被任何人赢的玩家（含只打了平局的）→ 全额拿回自己的最终比价。

伪代码：

```
对每个输家 L（stake = L 的最终比价）:
  winners = 所有 compareSplit 赢 L 的玩家
  按 (尾牌 desc, 头牌 desc) 排序，头尾全同者并入同一梯队
  remaining = stake
  for 梯队 in 梯队列表:
    tierCap = Σ(梯队成员各自的最终比价)   // 每人对 L 的封顶
    可吃 = min(remaining, tierCap)
    梯队内平分「可吃」，个人不超过自己的最终比价；
      有人先封顶，余量由梯队内其他人继续吃
    remaining -= 实际吃掉
    if remaining == 0: break
  remaining > 0 → 退还给 L
```

### 1.3 底池的归属

底池 = 底注 + 芒果 + 所有中途弃牌者支付的钱（+ 上局作废遗留）。

- 归**尾牌最大**的存活玩家，**不受封顶限制**，整锅全拿（哪怕他只敲了很小的数）；
- 尾牌并列最大时，先剔除被其他并列者两两击败的（头牌分胜负），剩余并列者平分；
- 即使全场互为平局、没有任何输赢，底池仍按"尾大吃皮"发放。

## 二、下注流程（喊价模型，定稿）

### 2.1 开局与发话顺序

- 只有庄家交底注 10 分（固定值），进底池；若上局触发芒果，全员打芒（揍芒赢家除外），进底池；
- 第 1 轮从庄家下家开始发话，逆时针依次；
- 第 2、3 轮由当轮明牌（第 3 / 第 4 张牌）最大者先发话（敲了的玩家也参与"最大者"判定，但轮到时自动跳过；多名并列最大时，按**第一轮发话顺序**谁在前谁先说——即庄家下家起的逆时针顺序）。

### 2.2 喊价模型（核心改动点二）

- 所有下注数字都是玩家**整局的喊价（累计总额）**，只会被抬高，不按轮清零。例：第 1 轮喊到 30 的玩家，第 2 轮"跟 60"是把整局喊价从 30 抬到 60（比牌时共付 60），不是 30 之外再付 60；
- 喊价期间筹码不离手；底池只包含：底注 + 芒果 + 弃牌者支付的钱（+ 上局作废遗留）；
- 弃牌 = 立即支付"**自己当前的喊价**"进底池（不是场上最高价），退出本局；
- 敲 = 喊价直接变为自己的全部簸簸数（之前喊的部分没实际扣过，所以敲出的是完整簸簸）；若高于当前最高喊价，视同加注；
- 比牌时各人按最终喊价（= 最终比价）入池，进入第一节的结算流程。

标准示例（甲庄，簸簸 甲100/乙120/丙80/丁300）：

| 阶段 | 过程 | 结果 |
|---|---|---|
| 开局 | 甲交底注 10 | 甲簸簸 90，底池 10 |
| 第1轮 | 乙休→丙喊10→丁加到30→甲跟30→乙跟30→丙弃牌付10 | 最终下注额 30，底池 20 |
| 第2轮 | 甲先发话（明牌最大），喊60（≥30×2）→乙敲120→丁跟120→甲簸簸90<120只能敲90 | 甲乙已敲，丁喊价120 |
| 第3轮 | 只剩丁一名未敲玩家 → 不再下注，直接发满牌 | 进入配牌比牌 |
| 结算 | 最终比价：甲90、乙120、丁120 | 按第一节结算 |

### 2.3 喊价额度规则

- 第 1 轮最低喊价 10；
- 第 2、3 轮的本轮最低喊价 = 上一轮最终下注额 × 2（约束本轮第一个下注的人）；
- 同一轮内的再加注：只需高于当前最高喊价即可，无翻倍等幅度要求；
- 加注无次数上限；任何喊价不得超过自己的簸簸数。

### 2.4 休与全员休

- 本轮没人抬价前都可以休；休 = 让后面的人先表态，轮回来后可正常跟/大/敲/丢；本轮有人抬价后不可休；
- **任何一轮**全员休 → 本局作废：存活玩家的喊价作废、分文不付；**弃牌者已付的钱退回本人**；只保留底注 + 芒果进入下一局底池；下一局全员打芒（休芒）。
- **整场结束时若底池仍有余额**：在仍占座且未淘汰的玩家中均分 `floor(底池/人数)`；**余数一律归当前庄家（不论庄家是否仍在场）**；无人可分时整锅归庄家。分配后底池清零，记入总结算。

### 2.5 敲（all-in）相关

- 敲的金额低于当前最高喊价：允许，继续参赛，不构成加注，其他人无需响应；
- 敲的金额高于当前最高喊价：等同加注，已跟过低价的玩家必须补齐/敲/丢；
- 早轮敲了的玩家不影响后续轮：其余玩家照常过招（翻倍规则继续），敲者被跳过；
- **只剩 ≤1 名未敲的存活玩家、且其喊价已跟平（或本就最高）时：不再进行任何后续下注轮，直接发满 4 张牌进入配牌比牌**。

### 2.6 UI 口径

- 底池数字只显示：底注 + 芒果 + 弃牌者支付（喊价不计入）；
- 每位玩家的当前喊价显示在各自座位旁。

## 三、三花、超时、断线、庄家与破产（定稿）

### 3.1 三花亮牌

- **时机**：只能在轮到自己发话时亮（第 3 张或第 4 张牌构成三花后的那次发话机会）；
- **一次性资格**：构成三花后的第一次发话机会没亮（选择了跟/大/敲等其他操作），即永久失去亮牌资格，当普通牌继续喊价、配牌、比牌。第 3 张构成三花时放弃了，第 4 张后也不能再亮；
- **自愿**：玩家觉得牌能赢可以主动放弃亮三花；
- **亮牌效果**：本局免输不赢——已喊的价作废、分文不付；已交的底注/芒果不退，留在底池；
- **不算弃牌**：亮三花退出本局**不构成弃牌**，即使本局已有人下过注，也不触发揍芒；
- **展示**：只把构成三花的三张牌展示给所有玩家，第 4 张牌不展示。

### 3.2 超时与断线

- 轮到发话后 60 秒不操作（时长暂定，30 秒发警告）：
  - 若本轮还没人抬价（他本可以休）→ **自动休**；
  - 否则 → **自动弃牌，并照常支付自己当前的喊价**进底池（超时不能逃掉该输的筹码）；
- 连续 3 分钟无操作 → 移出座位。**被移除玩家的筹码档案必须记录保留**（起簸与累计加簸总额、离开时簸簸余额），整场结束后参与总结算，保证全场筹码对得上；
- 断线不立即处理，轮到他时走上述超时规则，重连后恢复状态。

### 3.3 庄家轮转

- 每局结束后庄家逆时针轮转，**没有连庄**；全员休作废的局同样轮转；轮转跳过已离座/破产玩家；
- 庄家在局间起身离座：下一局的庄家 = 起身前庄家的下家。

### 3.4 破产与加簸

- **加簸**（新功能）：任意时刻（包括输光后）可发起，追加的筹码在**当前局结束、结算完成后**才加到簸簸上、下一局生效（防止玩家看牌好临时加簸）；
- 簸簸输光的玩家进入**等待加簸**状态；加簸后下一局自动回归；
- **下一局开始时**簸簸仍为 0 且未加簸 → 自动离座（离座后仍可随时加簸重新坐回）；
- 离座后在座玩家少于 2 名 → 整场提前结束并总结算；
- **两人局特殊处理**：只剩两名玩家游戏时，若一人输光，**暂停游戏并弹窗**，提供两个选项：「加簸继续」/「立即结算退出」，避免直接离座导致整场瞬间结束。

## 四、座位、换座与观战席（定稿，新功能）

### 4.1 座位与顺序的基本原则

- 共 8 个固定座位；**每局开始时按当前座位的物理位置（逆时针）重新推导玩家顺序**。例：当前顺序甲乙丙丁，乙换到甲的上家的空座，则新顺序为乙甲丙丁；
- 换座后发牌顺序、发话顺序、"近庄优先"的平局判定全部以新座位顺序为准；庄家身份跟人走。

### 4.2 换座（点击空座）的允许时机

仅以下两种情况允许换座：

1. 整场游戏开始前（房主点"开始游戏"之前）；
2. 整场进行中，该玩家当前不在对局里：本局已弃牌（等待本局结束期间），或上一局已结束、下一局尚未开始的间隙。

**防规避当庄限制**：玩家点击空座换座时，判断他是否是**下一局的庄家**（= 现任庄家按当前座位顺序的下家）。如果是，禁止换座并提示"下一局你当庄，当完庄后才能换座"。目的：防止玩家通过换座规避当庄交底注。

### 4.3 入座生效时机

- 坐下时如果下一局已经开始（对局进行中）：入座**不影响当前局**，该玩家在座位上观战，等本局结束、下局开始时正式加入；
- 坐下时处于局间空隙：下局直接加入。

### 4.4 观战席

- 后加入房间的玩家默认进入观战席；点击空座坐下后，按 4.3 的逻辑加入游戏；8 个座位全满时只能观战；
- 观战席可见：所有玩家的明牌、各玩家的喊价与下注动作、底池、对局信息（比牌结果、结算）、房间公共消息，以及后续要加入的弹幕消息；
- 观战席不可见：各玩家的暗牌（手牌）；
- 实现要点：公共状态广播给房间内所有连接（含观战者）；含手牌的私有状态只发给玩家本人。

### 4.5 局中退出房间

- 一局进行中玩家强行退出房间（或被 3 分钟无操作移除）：按**自动弃牌**处理，支付其当前喊价进底池，本局继续；
- 系统必须保留其筹码档案（起簸与累计加簸总额、离开时簸簸余额），整场结束总结算时计入。
