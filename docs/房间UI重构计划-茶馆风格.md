# 房间 UI 重构计划（茶馆风格）

> 目标：按已确认的静态视觉稿，重构房间页 UI / 交互，**不改游戏规则与结算语义**。  
> 视觉与交互唯一参考：`ui-mockups/style-tea.html`（入口：`ui-mockups/index.html`）  
> 风格已选定：**茶馆扯旋**（绿毡 + 木边 + 暖金）。霓虹稿 `style-neon.html` 仅作对照，不实施。  
> 术语与规则仍以 `docs/结算规则与改造方案.md`、`docs/游戏术语表.md` 为准；喊价 = `committed` 累计目标值。

---

## 〇、执行原则（给后续 AI）

1. **先静态后代码**：改正式代码前，以 `ui-mockups/style-tea.html` 为准；若产品再改静态稿，先同步静态稿再动 `client/`。
2. **只动展示与交互层**：优先改 `client/src/pages/RoomPage.tsx`、`client/src/pages/room/**`、`client/src/styles/room.css`（及 `mobile-game.css`）。除非 UI 缺字段，否则不改 `server/` 规则。
3. **保留现有数据流**：继续用 `useRoomGameController` 的状态与 socket 动作；重构是换布局/组件样式与操作手势，不是重写协议。
4. **移动端优先**：按手机竖屏（约 390–430 宽）验收；桌面可居中窄栏。
5. **分阶段交付**：按本文「实施阶段」顺序做，每阶段可独立验收；不要一次大爆炸改完。
6. **对照验收**：每阶段完成后，并排打开静态稿与真实房间页，逐项勾选「验收清单」。

## 一、视觉规范（茶馆）

### 1.1 色板与材质

| Token | 建议值 | 用途 |
|-------|--------|------|
| `--felt` / mid / deep | `#1a4d38` / `#246348` / `#0c241a` | 牌桌毡面渐变 |
| `--wood` / dark | `#8a5a32` / `#4a2e16` | 桌沿木边 |
| `--gold` | `#e0c06a` | 品牌字、簸数、主 CTA |
| `--ink` | `#f4f0e6` | 主文字 |
| `--muted` | `rgba(244,240,230,0.52)` | 次要说明 |
| `--timer` | `#2ee6c8` | 倒计时环/数字（**不要用金色**，避免与头像边混淆） |
| `--timer-warn` / `--timer-crit` | `#ff8a3d` / `#ff4d4d` | ≤40% / ≤20% 时间 |
| `--danger` | `#d46a58` | 丢、敲强调 |

字体：标题可用衬线（如 Noto Serif SC），正文 Noto Sans SC / PingFang SC。背景：深绿径向渐变 + 极轻噪点，避免纯平色。

### 1.2 牌桌形状与留白

- 形状：**大圆角矩形**（约 `border-radius: 40px`），不是椭圆。
- 左右内收约 **13%**，顶部约 **14%**，底部贴 HUD，减少桌与操作区空白。
- 木边多层描边 + 内阴影；桌心可淡印「扯旋」。
- **头像在桌外环带，牌在桌内**；最上座位不得压桌（静态稿已把顶边下移）。

### 1.3 扑克牌面

- 与现网 `CardView` 一致：点数 + 花色（♠♥♦♣），红黑配色；鬼牌单独样式。
- 他人桌面牌略小；自己手牌中等尺寸，可轻微扇形旋转。

## 二、布局结构

整体三行：`顶栏 | 舞台(桌+座位+牌) | 底部 HUD`。

```
┌─────────────────────────────┐
│ 房号 · 局数/规则     ☰ 菜单 │
├─────────────────────────────┤
│     [最上座位：名/头/簸]     │
│  ┌─────── 圆角矩形桌 ─────┐ │
│  │  各座位公开牌(桌内)     │ │
│  │         底池            │ │
│  └─────────────────────────┘ │
│  左右座位贴桌外两侧          │
├─────────────────────────────┤
│ [自己倒计时条]               │
│ 自己信息          提示文案   │
│        自己手牌              │
│      操作按钮区              │
└─────────────────────────────┘
```

### 2.1 顶栏

- 左：房间号 + 局数/底注等短 meta。
- 右：☰ 菜单（观战、对局记录、加簸、返回大厅等）。
- **观战席、历史不要常驻占桌面**，一律进菜单。

### 2.2 座位（他人）

结构自上而下固定：

1. **倒计时数字**（仅当前行动座位显示，青绿）
2. **玩家名**
3. **头像**（庄家角标；敲则火焰；行动中青绿环）
4. **簸簸数**

约束：

- 最多 8 座；自己固定在底部 HUD，不占环上「底中」位（或底中空出）。
- 倒计时数字在**名字上方**，禁止与簸数重叠。
- 火焰贴头像两侧/背后，**不得遮挡名字**（名字 `z-index` 高于火焰）。

### 2.3 桌面牌（他人）

- 牌锚在**桌面内侧**、靠近对应头像，不挂在头像下方。
- **下注阶段**：他人只显示可见的后两张（与现规则一致）；底牌不展示。
- **比牌阶段**：显示 4 张；**不要「头/尾」文字标签**；约定前两张=头、后两张=尾。
- 比牌排布默认 **两排（2×2）**；可选一排四张（产品可开关，静态稿有切换）。
- 弃牌座位可灰显，牌仍按规则展示。

### 2.4 底池

- 桌心：筹码叠 + 底池数字 + 副文案（最低叫分 / 当前喊价 / 比牌中）。

### 2.5 底部 HUD（自己）

自上而下：

1. **自己行动倒计时条**（仅轮到自己时）：左秒数 + 进度条 +「请操作」
2. 自己头像（可带小环）+ 名 + 簸数 | 右侧短提示
3. 自己手牌
4. 操作按钮区（随状态切换整块布局）

## 三、交互规范（必须按状态实现）

### 3.1 操作按钮布局（防乱套）

| 状态 | 布局 | 按钮 |
|------|------|------|
| 开叫（尚未有人抬价） | 左：休·丢；中：主 CTA「叫」；右：敲 | 与文档「第一轮可休」一致 |
| 已开叫（已有抬价） | 左：敲·丢；中：主 CTA「返」；右：瞧 | |
| 短筹码（只能敲/丢） | 两大按钮并排：敲 \| 丢 | |
| 配牌 | 重选 \| 确认 | 仅两键 |
| 比牌/等待 | 单按钮或文案：等待下一局 | |

实现建议：按状态切换整块 `ActionBar` 变体（`open` / `raised` / `short` / `split` / `compare`），不要同一排条件拼装导致错位。

### 3.2 叫分 / 返分（拖动选值）

- 主 CTA 显示：`叫 {n}` / `返 {n}`，`n` 为**整局累计喊价目标**（与后端 `committed` 一致）。
- **按住 + 上下拖**快速改值（上增下减）；松手后金额留在按钮上。
- **再点一次**提交；若几乎未拖动直接松手，可视为直接提交（静态稿行为）。
- 范围：`min`～自己簸数上限；步进与现网一致。
- 拖动时按钮可轻微放大，并显示「上下拖动改金额」提示。

> 替换现有横向滑条方案时，保持数值语义不变（见手动测试计划「滑条=累计目标」）。

### 3.3 配牌

1. 仅「重选」「确认」两按钮。
2. 点选手牌，最多 2 张为头；未满 2 张时「确认」禁用。
3. 确认后按钮变为 ✅（已确认态）；「重选/取消」清空选择并恢复可配。
4. 选中牌上移/描金边反馈。

### 3.4 倒计时

| 对象 | 展示 |
|------|------|
| 他人行动 | 头像青绿环 + 名字上方秒数；紧急变橙/红 |
| 自己行动 | HUD 顶部进度条 + 秒数；头像小环同步 |

倒计时颜色禁止用与头像描边相近的金色。

### 3.5 敲 → 火焰

- 已敲玩家：头像描边偏橙红 + **火焰动效**（CSS/轻量动画即可）。
- 火焰不得遮挡名字与倒计时数字。

### 3.6 发牌动画

- **前两张（底牌）**：不对他人做飞牌展示（可无动画或仅自己手牌静默出现）。
- **后两张（可见牌）**：必须动画；按座位**顺时针依次**发，一轮一张，共两轮（第 3、4 张）。
- 飞牌落点：他人 → 其桌面牌位；自己 → 手牌区对应张。
- 发牌期间可禁用操作 / 隐藏行动倒计时，结束后再进入行动态。

### 3.7 筹码动画

- 叫/返/敲/瞧等入池动作：从自己（或行动座位）飞若干筹码到桌心底池，底池数字更新。
- 可复用/改造现有 `AnimatedLayer` / Pixi 层；若成本高，先用 DOM + Web Animations，与静态稿一致即可。

## 四、建议改动的代码落点

| 区域 | 主要文件 | 改动要点 |
|------|----------|----------|
| 房间壳 | `RoomPage.tsx` | 三区布局；菜单收纳观战/历史；茶馆背景 token |
| 控制器 | `useRoomGameController.ts` | 暴露倒计时剩余、当前行动者、敲状态；叫/返提交仍走原 API；拖动手势状态可放此或 ActionBar |
| 座位 | `PlayerSeat.tsx` + `room.css` | 名/头/簸顺序；桌外定位；火焰；他人倒计时；牌与座位解耦 |
| 桌面牌 | `PublicCards.tsx` 或座位旁新组件 | 牌锚桌内；比牌去头尾标签；2×2 默认 |
| 手牌 | `MyHand.tsx` | 配牌点选、确认态样式 |
| 操作区 | `ActionBar.tsx` | 五态布局；叫/返拖动 CTA；短筹码双按钮；配牌确认✅ |
| 动画层 | `AnimatedLayer.tsx` / `PixiTableLayer.tsx` | 顺序发可见牌；飞筹码；评估是否弱化 Pixi 以免双轨 |
| 样式 | `room.css`、`mobile-game.css`、`cards.css` | 茶馆桌、座位环、HUD、timer 色；清理与新布局冲突的旧 absolute |

**不要做的事（本轮）：**

- 不改结算、座位规则、socket 事件名（除非缺 UI 字段再最小补丁）。
- 不实施霓虹风格。
- 不把观战/历史重新铺回桌面常驻条。

## 八、风险与注意

- **Pixi 与 DOM 双轨**：若 `PixiTableLayer` 与新 DOM 桌冲突，优先统一到一种实现，避免两套定位。
- **绝对定位座位**：8 人位需按「自己视角旋转」的现有逻辑映射到新百分比坐标，勿写死只适合静态演示的座位号。
- **安全区**：底部按钮与 `env(safe-area-inset-bottom)` 对齐，防止被 Home 条挡住。
- **与手动测试计划的关系**：规则/Bug 仍跟 `docs/手动测试问题修改计划.md`；本文只覆盖 UI 重构。两者并行时，先保证规则正确，再套新皮。
